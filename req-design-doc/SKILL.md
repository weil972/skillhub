# 需求设计文档生成器

## 目标

生成一份 Markdown 格式的需求设计文档，**编码智能体可直接阅读并开始编码**，无需额外澄清。

## 工作流程

### 第一步：收集与澄清需求

解析用户输入，提取所有需求，对每个需求识别：

- **显性需求** — 用户明确说出的
- **隐性需求** — 逻辑上必须但用户没提的
- **模糊点** — 可以有多种理解的
- **缺失细节** — 边界情况、异常处理、约束条件等

### 第二步：与用户确认

生成文档前，向用户展示结构化确认清单：

- 编号列出所有识别到的需求
- ⚠️ 标记歧义和假设
- ❓ 标记需要用户确认的问题
- 汇总范围边界（范围内 vs 范围外）

**必须等待用户回复**，所有 ❓ 项解决后或用户明确说"按假设继续"后，才进入下一步。

### 第三步：生成文档

按 [references/template.md](references/template.md) 模板生成 Markdown 文档。

**核心原则：文档必须是可直接实施的规格书。** 遵循以下规则：

1. **技术栈必须明确** — 语言、框架、版本、包管理器、构建工具，不留歧义
2. **文件结构必须定义** — 列出每个要创建/修改的文件及其用途
3. **每个功能需求都有验收标准** — 用可验证的断言描述，不用模糊表述
4. **API 接口完全定义** — 方法、路径、请求/响应格式、状态码
5. **数据模型完全规格化** — 字段名、类型、约束、默认值、关系
6. **依赖清单** — 精确到包名和版本
7. **实现顺序明确** — 编号任务列表，智能体按顺序执行
8. **边界情况和异常处理** — 每个需求说明无效输入、空数据、权限不足等情况的处理
9. **不写废话** — 每个章节都必须包含可操作的信息，无用的章节直接删除

### 第四步：保存与交付

- 保存为 `.md` 文件到用户指定位置，默认保存在工作区
- 告知用户文件保存位置
- 提议直接启动编码智能体来实现文档中的需求

## 启动 Claude Code

文档生成后，直接使用 `sessions_spawn` 启动 Claude Code 编码智能体，将文档作为任务上下文传递。参数如下：

```
sessions_spawn(
  runtime: "acp",
  agentId: "claude-code",
  mode: "run",
  task: "阅读需求设计文档 {path}，按照"实现任务清单"章节中描述的顺序实现所有任务，遵循文档中指定的技术栈、文件结构和验收标准。"
)
```

- 不需要询问用户是否启动，文档生成后直接启动
- 如果用户明确表示不需要编码，则跳过此步骤
