# 需求设计文档模板

本模板用于生成编码智能体可直接消费和实施的需求文档。

```markdown
# 需求设计文档：{项目名称}

**版本：** v1.0  
**日期：** {YYYY-MM-DD}  
**状态：** 草稿 / 已确认

---

## 1. 概述

### 1.1 目标
{一句话说明这个项目要做什么}

### 1.2 范围
- **范围内：**
  - {功能1}
  - {功能2}
- **范围外：**
  - {不包含的内容}

---

## 2. 技术栈

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 语言 | {如: Python} | {如: 3.11+} | |
| 框架 | {如: FastAPI} | {如: 0.100+} | |
| 数据库 | {如: PostgreSQL} | {如: 15} | |
| 包管理 | {如: poetry / pip} | | |
| 其他依赖 | | | |

**完整依赖列表：**
```
{包名}=={版本}  # 用途说明
```

---

## 3. 项目结构

```
{项目名}/
├── {文件路径}          # {文件用途}
├── {文件路径}          # {文件用途}
└── ...
```

---

## 4. 数据模型

### {模型名称}

| 字段 | 类型 | 必填 | 默认值 | 约束 | 说明 |
|------|------|------|--------|------|------|
| id | UUID | 是 | 自动生成 | 主键 | 主键 |
| {字段} | {类型} | {是/否} | {默认值} | {唯一/索引/非空} | {说明} |

**模型关系：**
- {模型A} → {模型B}：{一对一 / 一对多 / 多对多}

**数据库迁移：** {如: 使用 Alembic / Prisma / 自动迁移}

---

## 5. 接口定义

### {接口分组名称}

#### `{方法} {/api/路径}`

**说明：** {一句话描述}

**请求：**
```json
{
  "field1": "string",   // 说明
  "field2": 0           // 说明
}
```

**成功响应** `{状态码}`：
```json
{
  "field1": "string"
}
```

**错误响应：**

| 状态码 | 场景 | 响应体 |
|--------|------|--------|
| 400 | {场景} | `{"error": "{信息}"}` |
| 404 | {场景} | `{"error": "{信息}"}` |

（每个接口按此格式定义）

---

## 6. 功能需求

### FR-001：{需求名称}

- **优先级：** P0/P1/P2
- **描述：** {具体要做什么}
- **涉及文件：** `{文件1}`, `{文件2}`
- **验收标准：**
  1. 假设 {前提条件}，当 {执行操作} 时，预期 {结果}
  2. ...
- **边界情况：**
  - {输入为空时}：{处理方式}
  - {数据不存在时}：{处理方式}
  - {无权限时}：{处理方式}

---

## 7. 非功能需求

| 编号 | 类别 | 描述 | 验收标准 |
|------|------|------|----------|
| NFR-001 | 性能 | {描述} | {如: 接口响应 < 200ms (P95)} |
| NFR-002 | 安全 | {描述} | {如: 所有接口需鉴权} |
| NFR-003 | 兼容 | {描述} | {如: 支持 Chrome 100+} |

---

## 8. 业务流程

### 8.1 主流程
1. {步骤1} → `{涉及文件/函数}`
2. {步骤2} → `{涉及文件/函数}`
3. ...

### 8.2 异常处理

| 场景 | 触发条件 | 处理方式 | 用户感知 |
|------|----------|----------|----------|
| {场景} | {条件} | {处理} | {提示/行为} |

---

## 9. 配置与环境

| 环境变量 | 必填 | 默认值 | 说明 |
|----------|------|--------|------|
| {变量名} | 是/否 | {默认值} | {说明} |

---

## 10. 实现任务清单

编码智能体按以下顺序执行：

- [ ] 1. 初始化项目：{框架、依赖、配置文件}
- [ ] 2. {具体任务} → `{涉及文件}`
- [ ] 3. {具体任务} → `{涉及文件}`
- [ ] ...
- [ ] N. 端到端验证：{如何确认全部实现正确}

---

## 附录

{补充说明、参考文档等。如无则删除此节}
```

## 适配指南

- **小型项目：** 可将第4/5/6节合并为"功能与数据"一节
- **无接口（如命令行工具）：** 删除第5节，添加"命令行接口"章节，定义命令和参数
- **无数据库：** 删除第4节，简化第3节
- **已有代码库：** 在第3节前添加"现有代码分析"章节，区分需修改和需新建的文件
- **第10节（实现任务清单）始终保留** — 这是智能体执行的主要依据
